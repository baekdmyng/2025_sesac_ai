# Git 명령어 정리

## 워크 플로우

1. 폴더 이동: cd [폴더 경로] // finder 에서 폴더를 터미널로 드래그 하면 편함!
2. 빈 파일 생성: touch [생성하고 싶은 파일 이름].파일 타입(ex. txt, py, md) 
3. 상태 확인: git status // 빨간색: 추적 안 됨, 초록색: 제대로 담겨 있음
4. git add hello.txt (명령어에 < > 기호는 빼고 입력!)
5. 커밋: git commit -m "메시지" (메시지는 짧고 명확하게)
6. 깃허브 전송: git push origin main (로컬에서 서버로 업로드)

## 히스토리 및 관리
- ls -al: 현재 폴더의 모든 파일 목록 보기 (숨겨진 .git 폴더 확인 가능)
- git log --oneline --graph --all: 커밋 이력을 한 줄로, 그래프 형태로 예쁘게 보기 (나가기는 q)
- git sparse-checkout: 깃허브 서버는 유지하되 내 맥북에서만 특정 폴더 숨기기
 + init --cone: 기능 활성화
 + set [폴더명]: 해당 폴더만 보이게 설정
 + disable: 다시 모든 폴더 보이게 하기
